GA4 BigQuery 电商用户行为分析

基于 Google Merchandise Store 公开数据集的 SQL 漏斗、渠道归因与用户生命周期分析
将实习中 GA / Mixpanel 周报取数工作，向上扩展为"懂业务指标"，向下扩展为"能用 SQL 查询原始事件数据"

项目背景

实习期间负责 www.easesourcing.com 的 SEO 运营与数据周报：从 GA 取数、Mixpanel 漏斗每日记录、整理用户转化周报。实际工作中发现，真实业务漏斗跨越三层——GA 流量（访客）→ 前端点击（"Try for Free"按钮）→ 后端 Agent 行为（开聊），GA 看板无法灵活拆解。

因此用 Google 公开 GA4 数据集，以 SQL 复现一套标准分析流程，完成从运营执行到数据分析的能力迁移。

数据来源与漏斗说明

数据来源：bigquery-public-data.ga4_obfuscated_sample_ecommerce，为 Google Merchandise Store 三个月真实流量的脱敏混淆数据集。

分析过程中发现原始漏斗存在事件顺序异常（部分用户的 purchase 时间戳早于 begin_checkout），属于该公开数据集的已知噪声特征。

本分析采用有序时间戳约束漏斗：仅统计按顺序经历 view_item → add_to_cart → begin_checkout → purchase 各步骤的用户，剔除顺序异常记录，以还原更符合真实购买路径的转化结构。

SQL 查询（点击展开）

1. 每日活跃用户（DAU） GROUP BY + COUNT DISTINCT 
SELECT event_date, COUNT(DISTINCT user_pseudo_id) AS dau
FROM ga4_obfuscated_sample_ecommerce.events_*
WHERE _TABLE_SUFFIX BETWEEN '20201101' AND '20201130'
GROUP BY event_date ORDER BY event_date;
2. 新老用户分层 CTE + JOIN + 首次访问日期判定 
WITH first_visit AS (
  SELECT user_pseudo_id, MIN(event_date) AS first_date
  FROM ga4_obfuscated_sample_ecommerce.events_*
  GROUP BY user_pseudo_id
)
SELECT event_date,
  COUNT(DISTINCT CASE WHEN event_date = first_date
    THEN e.user_pseudo_id END) AS new_users,
  COUNT(DISTINCT CASE WHEN event_date > first_date
    THEN e.user_pseudo_id END) AS returning_users
FROM ga4_obfuscated_sample_ecommerce.events_* e
JOIN first_visit f USING(user_pseudo_id)
GROUP BY event_date ORDER BY event_date;
3. 渠道来源拆解 CTE + CASE WHEN 渠道归类 
WITH sessions AS (
  SELECT user_pseudo_id,
    LOWER(COALESCE(traffic_source.source, '(not set)')) AS source,
    LOWER(COALESCE(traffic_source.medium, '(not set)')) AS medium
  FROM ga4_obfuscated_sample_ecommerce.events_*
  WHERE event_name = 'session_start'
),
classified AS (
  SELECT *,
    CASE
      WHEN source = '(direct)' OR medium IN ('(none)', 'direct') THEN 'Direct'
      WHEN medium IN ('organic', 'seo') THEN 'Organic Search'
      WHEN medium IN ('cpc', 'ppc') THEN 'Paid Search'
      WHEN medium = 'referral' THEN 'Referral'
      ELSE 'Other'
    END AS channel_group
  FROM sessions
)
SELECT channel_group,
  COUNT(DISTINCT user_pseudo_id) AS users
FROM classified GROUP BY channel_group ORDER BY users DESC;
4. 总转化漏斗（有序时间戳约束） LAG 窗口函数 
WITH user_events AS (
  SELECT user_pseudo_id, event_name,
    MIN(event_timestamp) AS min_ts
  FROM ga4_obfuscated_sample_ecommerce.events_*
  WHERE event_name IN ('view_item','add_to_cart',
    'begin_checkout','purchase')
  GROUP BY user_pseudo_id, event_name
),
ordered AS (
  SELECT user_pseudo_id,
    MAX(CASE WHEN event_name='view_item' THEN min_ts END) AS t_view,
    MAX(CASE WHEN event_name='add_to_cart' THEN min_ts END) AS t_add,
    MAX(CASE WHEN event_name='begin_checkout' THEN min_ts END) AS t_checkout,
    MAX(CASE WHEN event_name='purchase' THEN min_ts END) AS t_purchase
  FROM user_events GROUP BY user_pseudo_id
  HAVING t_view < t_add AND t_add < t_checkout AND t_checkout < t_purchase
)
SELECT
  COUNT(*) AS view_item,
  COUNT(*) AS add_to_cart,
  COUNT(*) AS begin_checkout,
  COUNT(*) AS purchase
FROM ordered;
5. 分渠道转化漏斗 核心查询 多表 JOIN + 分渠道聚合 
WITH user_channels AS (
  SELECT DISTINCT user_pseudo_id, channel_group
  FROM sessions_with_classification
),
funnel_by_channel AS (
  SELECT c.channel_group,
    COUNT(DISTINCT CASE WHEN f.t_view IS NOT NULL THEN f.user_pseudo_id END) AS view_users,
    COUNT(DISTINCT CASE WHEN f.t_add IS NOT NULL THEN f.user_pseudo_id END) AS add_users,
    COUNT(DISTINCT CASE WHEN f.t_checkout IS NOT NULL THEN f.user_pseudo_id END) AS checkout_users,
    COUNT(DISTINCT CASE WHEN f.t_purchase IS NOT NULL THEN f.user_pseudo_id END) AS purchase_users
  FROM ordered_funnel f
  JOIN user_channels c USING(user_pseudo_id)
  GROUP BY c.channel_group
)
SELECT channel_group, view_users,
  ROUND(100.0 * purchase_users / view_users, 2) AS overall_cvr_pct
FROM funnel_by_channel ORDER BY view_users DESC;

数据可视化

图表 1：整体转化漏斗（有序时间戳约束）

view_item → add_to_cart 流失最严重（仅 20.44%），约 4/5 的浏览用户未加购。进入结算流程后完成率较高（55.76%），说明决策门槛主要在"加购"环节，而非支付环节。

图表 2：各渠道用户规模与整体转化率

Organic Search 流量最大，但整体转化率最低（4.15%）；Paid Search 流量最小，转化率与 Organic 接近（4.16%），但 checkout→purchase 为各渠道最高（57.59%）。

图表 3：分渠道各步骤转化率

各渠道 view→add 转化率普遍在 19–21%，差异有限；分化主要体现在 checkout→purchase 环节，Paid Search（57.59%）和 Referral（56.79%）明显高于 Direct（55.18%），说明付费和引荐来源的用户付款意愿更强。

业务洞察与优化方向

5.1 整体漏斗：流失集中在加购环节

view_item → add_to_cart 转化率仅 20.44%，是整个漏斗流失最严重的节点，约 4/5 的浏览用户没有加购。而一旦用户进入结算（begin_checkout），最终完成购买的概率达到 55.76%，说明支付意愿本身没有问题，问题出在"浏览 → 加购"的决策门槛上。

  • 商品详情页是核心优化战场：价格展示、评论数量、库存紧迫感（"仅剩 X 件"）、加购按钮的视觉权重
  • 对高浏览低加购的商品做单独分析，排查是定价问题还是信息不足

5.2 渠道质量：Paid Search 转化链路最强

Paid Search 的 checkout→purchase 转化率 57.59%，是五个渠道中最高的，说明付费搜索带来的用户付款意愿最强、决策最确定。但 view_to_add 仅 19.12%，是各渠道最低，说明部分广告关键词吸引的用户与商品匹配度不高。

  • 付费渠道预算值得维持甚至加大，但需要优化关键词定向，提升 view→add 阶段的匹配度
  • 可以用 view_to_add 作为广告组质量的判断指标，低于 18% 的广告组考虑暂停或调整落地页

5.3 Referral 渠道：引流质量被低估

Referral 的 overall_cvr 4.53% 和 checkout→purchase 56.79% 均高于 Organic Search，且 view_to_add（20.73%）也不低。这说明来自外链的用户整体质量较高——他们带着明确意图从第三方站点跳转而来，决策链路更短。

  • 外链建设 ROI 较高，值得持续投入
  • 对 Referral 来源做二级拆分（哪些域名带来转化更高的流量），优先维护高质量引荐来源

5.4 Organic Search：流量最大，但加购率最低

Organic Search 贡献了 22,508 名 view_item 用户，占总量约 37%，是最大的流量来源。但 view_to_add 仅 19.86%，是除 Paid Search 外最低的，overall_cvr 4.15% 也是最低。这与 SEO 流量的典型特征一致：自然搜索用户处于信息收集阶段，购买意图相对分散。

  • 针对高浏览量但低加购的商品页面，优化内容相关性和 CTA 设计
  • SEO 选题上优先布局商业意图词（"buy"、"best"、"review"），而非纯信息词，从源头提升流量质量

5.5 与实际业务经验的结合

以上分析框架直接来源于实习中的真实场景：在 easesourcing.com B 端 SaaS 产品运营工作中，用户从官网进入到 Agent 开聊的转化路径同样分为三层（GA 流量 → 前端点击 → 后端行为），与本项目的 view → add → checkout → purchase 漏斗结构一致。本 SQL 项目是对该分析能力的系统化复现和扩展。
