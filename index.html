GA4 BigQuery 电商用户行为分析

基于 Google Merchandise Store GA4 公开数据集，使用 BigQuery SQL 拆解电商用户获客质量、购买路径分流与关键流失节点，模拟真实运营场景中的指标定义、路径分析和优化判断。

用户路径分析渠道获客转化漏斗
数据来源：bigquery-public-data.ga4_obfuscated_sample_ecommerce，Google Merchandise Store 三个月（2020-11 至 2021-01）真实流量的公开脱敏数据集。

本项目将电商购买路径拆分为两类：
1. 加购路径：view_item → add_to_cart → begin_checkout → purchase（严格序列口径）
2. 未观察到加购的结算购买路径：购买所在 session 内有 begin_checkout → purchase，但未观察到 add_to_cart

路径判断窗口以购买所在 session 为单位。跨 session 的加购行为（如历史购物车回访）可能被归入路径二，属于三个月数据窗口的已知限制。
用户级 vs Session 级：路径分析以 user_pseudo_id 为单位，渠道分析以 session 为单位，两者侧重不同，数字不直接对齐。

模块一：渠道获客质量

渠道分析采用包容口径：判断每个 session 内是否发生核心漏斗动作，不区分具体购买路径。

各渠道 session → view_item 差异有限（约 20–22%），说明渠道大类不是解释后续流失的主要变量。后续分析重点转向用户进入商品页之后的行为选择。

模块二：用户卡在哪里

共 61,335 名触发过核心漏斗动作的用户，按其走到的最远漏斗节点分类。

最大流失发生在商品页：77% 的用户看了商品后没有进入任何后续转化动作。这是整个购买链路的主要卡点，也是优化的主战场。

模块五：优化建议

1. 商品页承接（最大优先级）

最大卡点：77% 用户在商品页离开。优化方向：加购 CTA（位置、样式、文案）、直接结算入口（Buy Now）、价格透明度、评分评论展示、库存紧迫感、配送政策、信任背书。不要强迫用户必须先加购——两条路径都是有效转化链路。

2. 购物车放弃（加购路径专属）

加购路径中约 60% 的用户未进入 checkout。优化方向：购物车召回（邮件 / 站内提醒）、免邮门槛提示、限时折扣、减少购物车干扰项。

3. Checkout 体验（两条路径共同）

运费展示时机（不要在最后一步才显示）、支付方式覆盖、结算步骤数量、登录要求、退换货说明、信任标识。

4. 埋点完整性

约 35% 的购买用户在购买 session 内未观察到 add_to_cart，部分可能来自 Buy Now、历史购物车回访或 add_to_cart 埋点未覆盖。建议确认这些路径的埋点完整性，并单独监控其 checkout 完成率。

SQL 附录（点击展开）

1. 路径分布总览（瀑布口径） 
WITH funnel_steps AS (
  SELECT user_pseudo_id,
    MIN(CASE WHEN event_name='view_item' THEN event_timestamp END) AS t_view,
    MIN(CASE WHEN event_name='add_to_cart' THEN event_timestamp END) AS t_add,
    MIN(CASE WHEN event_name='begin_checkout' THEN event_timestamp END) AS t_checkout,
    MIN(CASE WHEN event_name='purchase' THEN event_timestamp END) AS t_purchase
  FROM ga4.events_*
  WHERE event_name IN ('view_item','add_to_cart','begin_checkout','purchase')
  GROUP BY user_pseudo_id
)
SELECT -- 按最远到达节点分类
  CASE
    WHEN t_view IS NOT NULL AND t_view < t_add AND t_add < t_checkout AND t_checkout < t_purchase THEN '01 标准路径'
    WHEN t_purchase IS NOT NULL THEN '02 非标准路径（有purchase但顺序异常）'
    WHEN t_checkout IS NOT NULL AND t_purchase IS NULL THEN '03 停在 checkout'
    WHEN t_add IS NOT NULL AND t_checkout IS NULL THEN '04 停在 add_to_cart'
    WHEN t_view IS NOT NULL THEN '05 停在 view_item'
    ELSE '06 无 view 但有 add 或 checkout'
  END AS path_type,
  COUNT(DISTINCT user_pseudo_id) AS users
FROM funnel_steps GROUP BY 1 ORDER BY users DESC;
2. 购买路径分流 
SELECT
  COUNT(CASE WHEN t_view<t_add AND t_add<t_checkout AND t_checkout<t_purchase THEN 1 END) AS 加购路径购买,
  COUNT(CASE WHEN t_purchase IS NOT NULL AND
    NOT(t_view<t_add AND t_add<t_checkout AND t_checkout<t_purchase) THEN 1 END) AS 未观察到加购的结算购买,
  COUNT(CASE WHEN t_purchase IS NOT NULL AND t_view IS NULL AND t_add IS NULL THEN 1 END) AS 其他
3. 加购路径严格序列漏斗 
SELECT event_name, COUNT(DISTINCT user_pseudo_id) AS users
FROM ga4.events_*
WHERE event_name IN ('view_item','add_to_cart','begin_checkout','purchase')
  AND -- 仅统计严格按序经历的用户
GROUP BY event_name ORDER BY
  CASE event_name WHEN 'view_item' THEN 1 WHEN 'add_to_cart' THEN 2
  WHEN 'begin_checkout' THEN 3 WHEN 'purchase' THEN 4 END;
4. 渠道获客质量（包容口径） 
SELECT channel_group,
  COUNT(*) AS sessions,
  COUNT(DISTINCT CASE WHEN ARRAY_LENGTH(ARRAY(SELECT 1 FROM UNNEST(event_name) e WHERE e='view_item'))>0
    THEN user_pseudo_id END) AS view_item_sessions,
  COUNT(DISTINCT CASE WHEN ... purchase ... THEN user_pseudo_id END) AS purchase_sessions
FROM ga4.events_* GROUP BY channel_group;
5. SQL A：商品页后的下一步行为 
WITH session_events AS (
  SELECT user_pseudo_id,
    (SELECT value.int_value FROM UNNEST(event_params) WHERE key='ga_session_id') AS ga_session_id,
    event_name, event_timestamp
  FROM ga4.events_*
  WHERE event_name IN ('view_item','add_to_cart','begin_checkout','purchase')
),
classified AS (
  SELECT *,
    CASE
      WHEN t_add IS NOT NULL AND (t_checkout IS NULL OR t_add < t_checkout) THEN 'add_to_cart 路径'
      WHEN t_checkout IS NOT NULL AND (t_add IS NULL OR t_add >= t_checkout) THEN '直接进入 checkout'
      WHEN t_purchase IS NOT NULL AND t_add IS NULL AND t_checkout IS NULL THEN '直接 purchase'
      ELSE '无下一步核心动作'
    END AS next_action_type
  FROM view_sessions
)
SELECT next_action_type, COUNT(*) AS sessions,
  ROUND(100.0 * COUNT(*) / SUM(COUNT(*)) OVER(), 2) AS pct_of_view_sessions
FROM classified GROUP BY next_action_type ORDER BY sessions DESC;
6. SQL B：Checkout 入口完成率对比 
SELECT
  CASE WHEN t_add IS NOT NULL AND t_add < t_checkout THEN '加购后进入 checkout'
  ELSE '未观察到加购直接进入 checkout'
  END AS checkout_entry_type,
  COUNT(*) AS checkout_sessions,
  SUM(CASE WHEN t_purchase IS NOT NULL THEN 1 ELSE 0 END) AS purchase_sessions,
  ROUND(100.0 * SUM(CASE WHEN t_purchase IS NOT NULL THEN 1 END) / COUNT(*), 2) AS checkout_to_purchase_pct
FROM checkout_sessions GROUP BY checkout_entry_type ORDER BY checkout_sessions DESC;

分析口径与限制

渠道归因限制

使用 traffic_source.source / medium 作为渠道来源的近似拆解，非精确广告归因。公开数据集中 session 级 attribution 字段有限。

三个月数据窗口

数据覆盖 2020-11-01 至 2021-01-31。窗口外的行为不可见，部分跨 session 的加购记录会被归入"未观察到加购的结算路径"。

用户级 vs Session 级

路径分析以 user_pseudo_id 为单位；渠道分析以 session 为单位。两者侧重不同，数字不直接相加或对齐。

公开脱敏样本

基于 Google Merchandise Store 公开脱敏样本，适合探索性路径分析与 SQL 口径练习，不作为完整生产级归因模型。
